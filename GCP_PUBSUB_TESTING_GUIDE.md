# GCP Pub/Sub: Руководство по тестированию

## 🧪 Тестирование функциональности

### 1. CRUD операции для Topics

#### ✅ Тест 1.1: Создание Topic
**Шаги:**
1. Открыть компонент GCP Pub/Sub
2. Перейти на вкладку "Topics"
3. Нажать кнопку "Add Topic"
4. Проверить, что создан topic с именем "new-topic"
5. Проверить, что все поля инициализированы:
   - `name`: "new-topic"
   - `projectId`: текущий project ID или дефолтный
   - `messageRetentionDuration`: 604800 (7 дней)
   - `labels`: пустой объект `{}`
   - `messageCount`: 0
   - `byteCount`: 0
   - `publishedCount`: 0
6. Проверить toast-уведомление "Topic 'new-topic' created successfully"

**Ожидаемый результат:**
- ✅ Topic создан с корректными значениями по умолчанию
- ✅ Toast-уведомление отображается
- ✅ Topic отображается в списке

#### ✅ Тест 1.2: Создание Topic с дубликатом имени
**Шаги:**
1. Создать topic с именем "test-topic"
2. Попытаться создать еще один topic с именем "test-topic"
3. Проверить, что автоматически генерируется уникальное имя "test-topic-1"

**Ожидаемый результат:**
- ✅ Автоматическая генерация уникального имени
- ✅ Оба topic созданы с разными именами

#### ✅ Тест 1.3: Редактирование Topic
**Шаги:**
1. Создать topic "test-topic"
2. Изменить имя на "updated-topic"
3. Изменить `messageRetentionDuration` на 86400 (1 день)
4. Добавить label: key="environment", value="production"
5. Проверить, что изменения сохраняются

**Ожидаемый результат:**
- ✅ Имя изменено на "updated-topic"
- ✅ Retention duration изменен на 86400
- ✅ Label добавлен
- ✅ Изменения сохраняются корректно

#### ✅ Тест 1.4: Удаление Topic
**Шаги:**
1. Создать topic "test-topic"
2. Нажать кнопку удаления
3. Подтвердить удаление в диалоге
4. Проверить, что topic удален

**Ожидаемый результат:**
- ✅ Диалог подтверждения отображается
- ✅ Topic удален после подтверждения
- ✅ Toast-уведомление "Topic 'test-topic' deleted successfully"

#### ✅ Тест 1.5: Удаление Topic с зависимостями
**Шаги:**
1. Создать topic "test-topic"
2. Создать subscription, привязанную к "test-topic"
3. Попытаться удалить "test-topic"
4. Проверить сообщение об ошибке

**Ожидаемый результат:**
- ✅ Ошибка: "Cannot delete topic 'test-topic': 1 subscription(s) are using it"
- ✅ Topic не удален
- ✅ Toast-уведомление с ошибкой

### 2. CRUD операции для Subscriptions

#### ✅ Тест 2.1: Создание Subscription
**Шаги:**
1. Создать topic "test-topic"
2. Перейти на вкладку "Subscriptions"
3. Нажать "Add Subscription"
4. Проверить инициализацию полей:
   - `name`: "new-subscription"
   - `topic`: "test-topic" (первый доступный)
   - `projectId`: текущий project ID
   - `ackDeadlineSeconds`: 10
   - `enableMessageOrdering`: false
   - `filter`: { type: 'none' }
   - Все метрики: 0

**Ожидаемый результат:**
- ✅ Subscription создан с корректными значениями
- ✅ Toast-уведомление отображается

#### ✅ Тест 2.2: Создание Subscription без Topics
**Шаги:**
1. Убедиться, что нет topics
2. Попытаться создать subscription
3. Проверить сообщение об ошибке

**Ожидаемый результат:**
- ✅ Ошибка: "Cannot create subscription: no topics available"
- ✅ Subscription не создан

#### ✅ Тест 2.3: Редактирование Subscription
**Шаги:**
1. Создать subscription "test-sub"
2. Изменить имя на "updated-sub"
3. Изменить `ackDeadlineSeconds` на 30
4. Включить `enableMessageOrdering`
5. Добавить push endpoint: "https://api.example.com/push"
6. Проверить сохранение

**Ожидаемый результат:**
- ✅ Все изменения сохраняются
- ✅ Push endpoint отображается

#### ✅ Тест 2.4: Удаление Subscription
**Шаги:**
1. Создать subscription "test-sub"
2. Удалить subscription
3. Проверить удаление

**Ожидаемый результат:**
- ✅ Диалог подтверждения
- ✅ Subscription удален
- ✅ Toast-уведомление

### 3. Edge Cases

#### ✅ Тест 3.1: Пустые значения
**Шаги:**
1. Попытаться создать topic с пустым именем
2. Попытаться изменить имя topic на пустую строку
3. Попытаться создать subscription с пустым именем
4. Попытаться изменить topic subscription на пустую строку

**Ожидаемый результат:**
- ✅ Ошибка валидации для пустых имен
- ✅ Toast-уведомление с описанием ошибки
- ✅ Изменения не сохраняются

#### ✅ Тест 3.2: Дубликаты имен
**Шаги:**
1. Создать topic "test-topic"
2. Попытаться создать еще один topic "test-topic"
3. Попытаться изменить имя другого topic на "test-topic"
4. Повторить для subscriptions

**Ожидаемый результат:**
- ✅ Автоматическая генерация уникального имени при создании
- ✅ Ошибка валидации при попытке изменить имя на существующее
- ✅ Toast-уведомление "Name must be unique"

#### ✅ Тест 3.3: Некорректные индексы
**Шаги:**
1. Создать несколько topics
2. Удалить topic с индексом 0
3. Попытаться обновить topic с индексом, который больше не существует
4. Попытаться удалить topic с отрицательным индексом

**Ожидаемый результат:**
- ✅ Валидация индекса перед операциями
- ✅ Ошибка: "Invalid topic index" или "Topic not found"
- ✅ Toast-уведомление с ошибкой

#### ✅ Тест 3.4: Некорректные числовые значения
**Шаги:**
1. Попытаться установить `ackDeadlineSeconds` < 10
2. Попытаться установить `ackDeadlineSeconds` > 600
3. Попытаться установить `messageRetentionDuration` < 600
4. Попытаться установить `messageRetentionDuration` > 2678400
5. Попытаться установить отрицательные значения

**Ожидаемый результат:**
- ✅ Валидация диапазонов
- ✅ Ошибки валидации с описанием допустимых значений
- ✅ Toast-уведомления

#### ✅ Тест 3.5: Некорректные форматы имен
**Шаги:**
1. Попытаться создать topic с именем "Test-Topic" (заглавные буквы)
2. Попытаться создать topic с именем "test topic" (пробелы)
3. Попытаться создать topic с именем "test@topic" (спецсимволы)
4. Попытаться создать topic с именем "ab" (слишком короткое)
5. Попытаться создать topic с именем длиннее 255 символов

**Ожидаемый результат:**
- ✅ Валидация формата имен
- ✅ Ошибки с описанием правил именования
- ✅ Toast-уведомления

#### ✅ Тест 3.6: Изменение topic subscription на несуществующий
**Шаги:**
1. Создать subscription, привязанную к "topic-1"
2. Удалить "topic-1"
3. Попытаться изменить topic subscription на "topic-1"
4. Попытаться создать subscription с несуществующим topic

**Ожидаемый результат:**
- ✅ Валидация существования topic
- ✅ Ошибка: "Topic 'topic-1' does not exist"
- ✅ Toast-уведомление

### 4. Фильтры подписок

#### ✅ Тест 4.1: Создание subscription с attribute filter
**Шаги:**
1. Создать subscription "test-sub"
2. Изменить filter type на "attributes"
3. Добавить атрибут: key="environment", value="production"
4. Добавить еще один атрибут: key="priority", value="high"
5. Проверить сохранение

**Ожидаемый результат:**
- ✅ Filter type изменен на "attributes"
- ✅ Атрибуты добавлены и отображаются
- ✅ Изменения сохраняются

#### ✅ Тест 4.2: Редактирование filter attributes
**Шаги:**
1. Создать subscription с filter attributes
2. Изменить значение атрибута "environment" на "staging"
3. Удалить атрибут "priority"
4. Добавить новый атрибут "region" = "us-east-1"

**Ожидаемый результат:**
- ✅ Изменения атрибутов сохраняются
- ✅ Удаление работает
- ✅ Добавление работает

#### ✅ Тест 4.3: Переключение filter type
**Шаги:**
1. Создать subscription с attribute filter
2. Изменить filter type на "none"
3. Проверить, что атрибуты скрыты
4. Изменить обратно на "attributes"
5. Проверить, что можно добавить атрибуты

**Ожидаемый результат:**
- ✅ Переключение работает
- ✅ UI обновляется корректно
- ✅ Атрибуты сохраняются при переключении

#### ✅ Тест 4.4: Применение фильтров в симуляции
**Шаги:**
1. Создать topic "test-topic"
2. Создать subscription "sub-1" с filter: environment=production
3. Создать subscription "sub-2" с filter: environment=staging
4. Запустить симуляцию
5. Отправить сообщение в topic с атрибутом environment=production
6. Проверить, что сообщение доставлено только в sub-1

**Ожидаемый результат:**
- ✅ Фильтры применяются при доставке
- ✅ Сообщения с неподходящими атрибутами не доставляются
- ✅ Метрики отражают правильную доставку

### 5. Dead Letter Topics

#### ✅ Тест 5.1: Настройка Dead Letter Topic
**Шаги:**
1. Создать topic "main-topic"
2. Создать topic "dlq-topic"
3. Создать subscription "test-sub" на "main-topic"
4. Настроить Dead Letter Topic: "dlq-topic"
5. Установить maxDeliveryAttempts: 3
6. Проверить сохранение

**Ожидаемый результат:**
- ✅ Dead Letter Topic выбран
- ✅ Max Delivery Attempts установлен
- ✅ Настройки сохраняются

#### ✅ Тест 5.2: Перемещение сообщений в DLQ
**Шаги:**
1. Создать topic "main-topic" и "dlq-topic"
2. Создать subscription с DLQ и maxDeliveryAttempts=2
3. Запустить симуляцию
4. Отправить сообщение, которое не будет обработано (expired)
5. Проверить, что после 2 попыток сообщение перемещено в dlq-topic
6. Проверить атрибуты сообщения в DLQ:
   - x-original-subscription
   - x-delivery-attempts
   - x-failed-reason

**Ожидаемый результат:**
- ✅ Сообщения перемещаются в DLQ после превышения попыток
- ✅ Атрибуты добавлены корректно
- ✅ Метрики отражают перемещение

#### ✅ Тест 5.3: Изменение maxDeliveryAttempts
**Шаги:**
1. Создать subscription с maxDeliveryAttempts=5
2. Изменить на maxDeliveryAttempts=1
3. Проверить валидацию (1-100)
4. Попытаться установить 0 или 101

**Ожидаемый результат:**
- ✅ Валидация диапазона 1-100
- ✅ Ошибка при некорректных значениях
- ✅ Toast-уведомления

#### ✅ Тест 5.4: Удаление Dead Letter Topic
**Шаги:**
1. Создать subscription с DLQ
2. Удалить Dead Letter Topic из настроек
3. Проверить, что настройка сброшена

**Ожидаемый результат:**
- ✅ DLQ можно удалить
- ✅ Настройка сбрасывается корректно

## 📝 Примечания

- Все тесты должны выполняться в браузере с открытой консолью для проверки ошибок

- Проверять работу в режиме симуляции для тестов фильтров и DLQ
